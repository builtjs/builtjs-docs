# Introduction

If you're designing Built.js themes or plugins to be used by the community, following these guidelines will help you keep conflicts between plugins and themes to a minimum and make designing easier.

These guidelines aim to follow industry best practices, allow for creative freedom and not be overly opinionated.

Community themes and plugins should:
1. Use [Tailwind](https://tailwindcss.com/) for styles
2. Have definitions in `tailwind.config` for primary, secondary, and accent colors as per the [Colors Design Guidelines](/community/design-guidelines/colors)
3. Use @apply for common typography, border and corner radius styles
4. Have classes sorted so they are ordered consistently. You can use [Headwind](https://github.com/heybourn/headwind) or similar for this
